# Fresh Adaptive Movement Methods

This stage adds two independent candidates to the existing method × training-window tournament.

## Bayesian Adaptive Memory

Internal identifier:

```text
bayesian_change_point
```

The method does not commit to one effective memory length inside its selected tournament window. It evaluates candidate run lengths of 7, 14, 28, 56, and 112 results, bounded by the training data available at prediction time.

For each candidate run length it builds a recency-weighted state and transition distribution, then calculates walk-forward predictive evidence using only results available before each calibration target. Posterior-like run-length weights combine:

- predictive likelihood on recent targets;
- a geometric change hazard;
- measured distribution and movement shift;
- preference for short memory when change pressure is high;
- preference for long memory when recent behavior is stable;
- bounded information quality of the resulting distribution.

The final digit distribution is the normalized mixture of all eligible run-length hypotheses. No future result is used.

## Adaptive Contextual Fusion

Internal identifier:

```text
adaptive_contextual_fusion
```

The method combines six independent signals:

- recency-weighted frequency;
- delta continuation;
- movement motif similarity;
- recurrence-cycle pressure;
- cross-position analog similarity;
- delta-transition behavior.

Initial signal priors depend on the current position regime: trend, zigzag, reversal, stable, or chaotic. The method then replays up to five recent calibration targets. For every target, each signal predicts from the history available before that target.

Signal weights are updated from a composite probability loss:

- multiclass Brier loss;
- top-three miss penalty;
- mild forgetting so recent evidence matters more.

The live signal distribution is additionally adjusted by information quality. Every signal retains an exploration floor, and one signal is capped at 42% of fusion weight. This prevents a brief streak from permanently collapsing the mixture into one component.

## Engine integration

Both methods are full base candidates. They automatically participate in:

- every training window generated by the Movement tournament;
- fixed L14 walk-forward evaluation;
- progressive L21, L28, L35, and later tie resolution;
- Walk-Forward Weighted Ensemble source weighting;
- live shadow candidate recording;
- persistent live weight settlement and decay;
- single-market and Batch Adaptif execution.

Candidate totals for 168 results become:

- Position, 3D, and 4D targets: 121 candidates;
- two-position targets: 132 candidates.

The count includes one Walk-Forward Weighted Ensemble candidate per window.

The migration expands the persistent live-weight method constraint. Without it, the prediction engine still returns output because live-weight loading is fail-safe, but settled shadow rows for the two new methods cannot be inserted into `adaptive_live_model_weights`.

## Safety properties

- Prediction for target `t` uses only history before `t`.
- Both methods remain candidates; neither bypasses the tournament.
- Results continue to be published under the existing always-publish behavior.
- Persistent live weighting remains bounded at its existing maximum contribution.
- Existing methods are not removed or renamed.
- All probability distributions are normalized, finite, and non-negative.
